Drag & Drop Library for the Scheduling Grid

Choose the library that will power card dragging in the assignment grid

A

@dnd-kit/core (Recommended)

Modern, lightweight, React-first DnD library

  • Built for React 18/19 — uses hooks, no legacy APIs
  • Excellent keyboard & screen reader accessibility (ARIA live regions)
  • Sortable, droppable, and draggable primitives compose cleanly
  • Custom sensors (pointer, keyboard, touch) — great for grid UX
  • Lightweight (~12KB gzipped) with tree-shaking
  • Active maintenance, good TypeScript support
  • Snap-back animation built in for invalid drops

Pros

  • Best React integration
  • Accessible by default
  • Performant with many items
  • Custom drag overlays

Cons

  • Need to build grid-specific logic ourselves
  • Smaller ecosystem than react-dnd
B

react-dnd

Battle-tested, flexible DnD library

  • Mature library with large ecosystem and community
  • Uses HTML5 backend (native drag) or touch backend
  • Drag source / drop target pattern is well-documented
  • Can handle complex multi-type drag scenarios
  • ~15KB gzipped

Pros

  • Well-documented, lots of examples
  • Handles complex scenarios
  • Stable API

Cons

  • Older API pattern (HOCs/hooks mix)
  • HTML5 backend has RTL issues
  • Less accessible out of box
  • Heavier setup boilerplate
C

@atlaskit/pragmatic-drag-and-drop

Atlassian's new DnD — powers Jira & Trello

  • Framework-agnostic core with React adapter
  • Built for board/grid layouts exactly like ours
  • Native drag events — best perceived performance
  • Tiny core (~4.7KB), add what you need
  • Production-proven at Atlassian scale

Pros

  • Purpose-built for boards/grids
  • Smallest bundle size
  • Battle-tested at scale (Jira)

Cons

  • Newer, smaller community
  • Less React-specific documentation
  • More manual setup for custom UX

My Recommendation: @dnd-kit/core

Best balance of React-native API, accessibility (important for hospital software), RTL support via CSS transforms (not HTML5 drag which has RTL bugs), and custom drag overlays for our assignment cards. The snap-back animation for invalid drops is exactly what the spec requires.
