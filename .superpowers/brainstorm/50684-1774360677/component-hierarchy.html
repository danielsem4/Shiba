Component Hierarchy

How the scheduler feature decomposes into components

SchedulerPage                          ← Page component (route entry)
│
├─ SchedulerToolbar                     ← Top action bar
│  ├─ Button: "שיבוץ ידני" (Manual)      → opens ManualAssignmentDialog
│  └─ Button: "ייבוא מאקסל" (Import)     → opens ExcelImportDialog
│
├─ SchedulerFilters                     ← Filter bar + legend
│  ├─ WeekRangeSelector                  ← Semester week range picker
│  ├─ UniversityFilter                   ← Dropdown multi-select
│  ├─ ShiftFilter                        ← Morning / Evening toggle
│  ├─ YearFilter                         ← Year in program dropdown
│  └─ AssignmentLegend                 ← Color key for card types
│
├─ DndContext (from @dnd-kit/core)        ← Wraps entire grid
│  │
│  ├─ SchedulerGrid                    ← CSS Grid container
│  │  │
│  │  ├─ GridHeader                    ← Sticky top row
│  │  │  ├─ DepartmentColumnHeader       ← "מחלקות" label (sticky right)
│  │  │  └─ WeekColumnHeader × N         ← "שבוע 1 (2/3-6/3)" per week
│  │  │
│  │  └─ GridRow × N                   ← One per department
│  │     ├─ DepartmentLabel              ← Sticky right department name
│  │     └─ GridCell × N               ← Droppable zone (dept × week)
│  │        ├─ [BlockedOverlay]          ← If constraint blocks this slot
│  │        └─ AssignmentCard × 0-N    ← Draggable cards
│  │           ├─ UniversityName
│  │           ├─ YearBadge
│  │           ├─ TypeBadge (Group/Elective)
│  │           └─ StudentCount
│  │
│  └─ DragOverlay                      ← Floating card while dragging
│
├─ ManualAssignmentDialog               ← Create new assignment form
│  ├─ DepartmentSelect
│  ├─ DatePickers (Sunday-only start, Thursday-only end)
│  ├─ UniversitySelect
│  ├─ AssignmentTypeSelect (Group/Elective)
│  ├─ NumberOfStudentsInput
│  ├─ YearInProgramSelect
│  └─ TutorNameInput
│
├─ ExcelImportDialog                   ← Drag & drop file upload
│  ├─ DropZone (drag file or browse)
│  ├─ ValidationStatus
│  └─ ColumnMappingPreview
│
└─ EditAssignmentDialog                ← Edit existing + student list
   ├─ AssignmentForm (same fields as Manual)
   └─ StudentListSection
      ├─ StudentTable (name, ID, phone, email)
      ├─ AddStudentRow (inline form)
      └─ ImportStudentExcel (mini dropzone)

Key Design Decisions

  • DndContext wraps only the grid — dialogs are outside the drag scope
  • GridCell is the droppable unit — each cell knows its departmentId + weekNumber
  • AssignmentCard is the draggable — carries its assignmentId as data
  • DragOverlay renders a ghost card that follows the cursor (not the original card)
  • BlockedOverlay renders on top of cells with constraints, with tooltip explaining why
  • All dialogs are lazy-loaded — only mount when opened
