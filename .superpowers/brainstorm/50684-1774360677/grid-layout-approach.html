Grid Layout Strategy

How should we build the departments × weeks scheduling grid?

A

CSS Grid + Sticky Headers (Recommended)

Native CSS Grid with custom scrolling

  • CSS display: grid with position: sticky for frozen department column
  • Department names stick on the right (RTL) while weeks scroll horizontally
  • Week headers stick on top while departments scroll vertically
  • Each cell is a droppable zone containing 0-N assignment cards
  • Tailwind classes for responsive sizing

Pros

  • Native browser performance
  • RTL works with CSS direction
  • Full control over cell rendering
  • Integrates naturally with dnd-kit

Cons

  • Manual sticky header implementation
  • Need custom scroll sync
B

TanStack Table (Virtual)

Headless table with built-in virtualization

  • TanStack Table for data structure + TanStack Virtual for row virtualization
  • Handles column pinning (for frozen department column) out of the box
  • Built-in sorting, filtering infrastructure
  • Already in the TanStack ecosystem (we use React Query)

Pros

  • Column pinning built-in
  • Virtualization for 50+ rows
  • Consistent with React Query usage

Cons

  • Table paradigm doesn't fit multi-card cells well
  • DnD integration requires extra work
  • Overhead for a visual grid that isn't a data table
C

Hybrid: CSS Grid + Virtual Rows

CSS Grid layout with optional row virtualization

  • CSS Grid for the visual layout (full control)
  • Add @tanstack/react-virtual only if performance needs it (20-50 rows may not)
  • Start simple, add virtualization as an optimization later
  • Best of both worlds — clean layout with escape hatch for scale

Pros

  • Start simple, optimize later
  • Full visual control
  • No premature complexity

Cons

  • May need refactoring if virtualization is added
